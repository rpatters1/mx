# mx::api Feature Audit

Generated by the `api-feature-audit` skill, 2026-06-29.
Corpus: 831 files (442 wild, 389 synthetic); 442 elements (326 used in the wild, 116
synthetic-only). `mx::api` audit: 345 features recorded in `data/api.features.xml`.

Tracking status and issue/PR numbers for every item are in the Tracker table at the bottom of this
doc. The parent tracker issue is #159 (with live checkboxes and sub-issues).

## How to read the severity

- BUG = silent data loss: a value `mx::core` (and the spec) can hold that `mx::api` maps to a
  fallback, so it is dropped on read and cannot be written back.
- GAP = an element/attribute `mx::api` does not model. Often by design; a gap in a heavily-used
  wild feature is a candidate to add.

Implementation path for everything here is the `add-mx-api-feature` skill. Do not implement from
this report.

## 1. BUG FIXES (enum members silently dropped)

All 30 `Converter::EnumMap` tables in `src/private/mx/impl/Converter.cpp` were audited.
Most previously-reported bugs have been fixed. Three remain.

### 1.1 `AccidentalValue::other` - LOW URGENCY

- Elements: `<accidental>` (wild-files=148), `<accidental-mark>` (wild-files=2)
- Core: `core::AccidentalValue::other()` (`src/private/mx/core/generated/AccidentalValue.h:114`)
- API: `api::Accidental` (`src/include/mx/api/PitchData.h:25`) has no `other` member
- Table: `Converter::accidentalMap` (`src/private/mx/impl/Converter.cpp:81`) - `other` absent
- Fallback: `api::Accidental::none`
- Same gap in `accidentalMarkMap` (`Converter.cpp:229`) - no `api::MarkType::accidentalMarkOther`
- Wild hit: no wild file currently uses `AccidentalValue::other`

Fix: add `api::Accidental::other` to `PitchData.h` and one row to `accidentalMap`. For
accidental-mark, add `api::MarkType::accidentalMarkOther` to `MarkData.h` and one row to
`accidentalMarkMap`. See `add-mx-api-feature` skill for the pattern.

### 1.2 Technical: `hammerOn`, `pullOff`, `tap` - MEDIUM URGENCY

- Element: `<technical>` (wild-files=18)
- Wild examples: `foundsuite/Yeah.xml`, `ksuite/k004a_Technical.xml`
- Core: `core::TechnicalChoice::Kind::{hammerOn, pullOff, tap}` (`TechnicalChoice.h:57-60`)
- API: `api::MarkType` has these members commented out (`MarkData.h:148-151`)
- Table: `Converter::technicalMarkMap` (`Converter.cpp:288`) - all three commented out
- Fallback: `api::MarkType::unspecified`
- Note: `hammerOn`/`pullOff` carry `start`/`stop` + text (slur-like pairing); `tap` carries text.
  Simple presence-mapping loses the pairing; a struct would be correct but more work.

Fix: design decision needed. Minimal: map hammerOn/pullOff start-only to flat mark types and accept
loss of the paired stop. Tap is simpler (text mark). See commented entries in `MarkData.h` and
`Converter.cpp:304-310`.

## 2. HIGH-USE GAPS (common in the wild, not fully modeled)

Ranked by wild-file count.

### 2.1 `<sound>` - wild-files=141 - HIGH VALUE

- Examples: `custom/systems-and-pages.xml`, `foundsuite/Adagio and Fugue in C minor, K.546.xml`
- `MeasureReader::parseSound` is a no-op stub (`MeasureReader.cpp:618`); `DirectionData.h:96` has
  a `// TODO - sound element` comment
- `<sound>` carries tempo, dynamics, MIDI playback cues, segno/coda targets, dacapo/fine markers.
  It is the primary playback-information element in MuseScore/Finale/Dorico exports.
- Effort: medium. A minimal `SoundData` covering `tempo` and repeat-navigation flags would cover
  the highest-value cases.

### 2.2 `<word-font>` - wild-files=129 - MEDIUM VALUE

- Examples: `custom/musescore-slur-start-stop.musicxml`, `custom/systems-and-pages.xml`
- `<defaults>/<word-font>` sets the default font for text directions. Dropped on read/write.
- Effort: small. Add a `wordFont` field to `DefaultsData`; wire in `LayoutFunctions.cpp`.

### 2.3 `<lyric-font>` - wild-files=109 - MEDIUM VALUE

- Examples: `custom/musescore-slur-start-stop.musicxml`, `foundsuite/Invention 1.xml`
- `<defaults>/<lyric-font>` sets the default font for lyrics. Dropped on read/write.
- Effort: small. Parallel to `word-font`; add to `DefaultsData`.

### 2.4 `<music-font>` - wild-files=91 - LOW VALUE

- `<defaults>/<music-font>` names the SMuFL music font. Dropped on read/write.
- Effort: small. Low priority unless font-specific rendering is planned.

### 2.5 `<source>` - wild-files=26 - LOW VALUE

- Part of `<identification>`; not read or written (`ScoreReader.cpp:179-240`).
- Effort: trivial. Add a `source` string field alongside `relation`.

## 3. ROUND-TRIP CLOSERS

Closing `word-font` + `lyric-font` (both small efforts) would make the defaults block complete
for the ~130 files that use only those two missing defaults features.

The dominant single-feature blocker across the whole corpus is `<sound>` (141 files): a file that
is otherwise fully supported but carries a `<sound>` element currently fails to round-trip on that
element alone.
